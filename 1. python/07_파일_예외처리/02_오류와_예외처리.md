## 오류와 예외 처리

### 오류의 종류

### Syntax Error

	- 문법 오류이다.
	- 실행 전에 코드 문법이 잘못되었을 때 발생한다.

	예제

```python
if a < 3
    print(a)
```

	오류 이유

```text
if 조건문 끝에 콜론(:)이 없다.
```

### Logical Error / Exception

	- 실행 중 발생하는 오류이다.

	예제

```python
a = 1 / 0
```

	오류

```text
ZeroDivisionError: division by zero
```

### 주요 내장 예외

| 예외 | 발생 상황 |
|---|---|
| SyntaxError | 문법 오류 |
| ZeroDivisionError | 0으로 나누기 |
| NameError | 정의되지 않은 이름 사용 |
| TypeError | 잘못된 자료형 연산 |
| ValueError | 값이 부적절함 |
| IndexError | 리스트 인덱스 범위 초과 |
| KeyError | 딕셔너리 Key 없음 |
| FileNotFoundError | 파일 없음 |
| AttributeError | 객체에 없는 속성 접근 |
| ImportError | 모듈 import 실패 |

### try except

	- 오류가 발생할 수 있는 코드를 안전하게 처리한다.

```python
try:
    result = 10 / 0
except ZeroDivisionError:
    print("0으로 나눌 수 없습니다.")
```

### 여러 예외 처리

```python
try:
    num = int(input("숫자 입력: "))
    result = 10 / num
except ValueError:
    print("숫자만 입력하세요.")
except ZeroDivisionError:
    print("0으로 나눌 수 없습니다.")
```

### 예외 정보 확인

```python
try:
    result = 10 / 0
except Exception as e:
    print(type(e))
    print(e)
```

### else

	- 예외가 발생하지 않았을 때 실행된다.

```python
try:
    num = int(input("숫자 입력: "))
except ValueError:
    print("잘못된 입력")
else:
    print("입력 성공:", num)
```

### finally

	- 예외 발생 여부와 관계없이 항상 실행된다.

```python
try:
    f = open("test.txt", "r")
except FileNotFoundError:
    print("파일이 없습니다.")
finally:
    print("프로그램 종료")
```

### raise

	- 사용자가 직접 예외를 발생시킨다.

```python
age = -1

if age < 0:
    raise ValueError("나이는 음수가 될 수 없습니다.")
```

### 사용자 정의 예외

```python
class CustomError(Exception):
    pass

raise CustomError("사용자 정의 오류 발생")
```

### 예제: 급여 범위 예외

```python
class SalaryNotInRangeError(Exception):
    """급여가 허용 범위를 벗어났을 때 발생하는 예외"""
    pass

salary = int(input("급여 입력: "))

if not 5000 <= salary <= 15000:
    raise SalaryNotInRangeError("급여 범위를 벗어났습니다.")
else:
    print("정상 급여")
```
