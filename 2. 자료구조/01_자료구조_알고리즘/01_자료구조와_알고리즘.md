## 자료구조와 알고리즘

### 자료구조(Data Structure)

	- 컴퓨터에서 자료를 정리하고 조직화하는 방법
	- 자료의 특성에 따라 효율적인 저장 방식이 다르다.
	- 프로그램은 보통 **자료구조 + 알고리즘**으로 구성된다.

	자료구조의 큰 분류

```text
자료구조
├─ 단순 자료구조
│  ├─ 숫자
│  └─ 문자
└─ 복합 자료구조
   ├─ 선형 자료구조
   │  ├─ 리스트
   │  ├─ 스택
   │  ├─ 큐
   │  └─ 덱
   └─ 비선형 자료구조
      ├─ 트리
      └─ 그래프
```

	선형 자료구조

	- 자료가 일렬로 나열되는 구조
	- 앞뒤 관계가 명확하다.

	예시

```text
A → B → C → D
```

	비선형 자료구조

	- 자료가 일렬로만 표현되지 않는 구조
	- 계층 또는 연결 관계를 표현한다.

	예시

```text
트리, 그래프
```

	시험 포인트

	Q. 다음 중 비선형 자료구조는?

	1. 리스트
	2. 큐
	3. 스택
	4. 그래프

	정답: 4

### 알고리즘(Algorithm)

	- 문제를 해결하기 위한 절차, 방법, 명령어의 모음
	- 프로그래밍 언어와 무관하게 설명 가능하다.

	알고리즘의 조건

	- 입력: 0개 이상
	- 출력: 1개 이상
	- 명백성: 각 단계가 명확해야 함
	- 유한성: 반드시 끝나야 함
	- 유효성: 실제 실행 가능한 명령이어야 함

	예시: 두 수 중 큰 수 찾기

```python
a = 10
b = 20

if a > b:
    print(a)
else:
    print(b)
```

	실행 결과

```text
20
```

	주의사항

	- 알고리즘은 무조건 복잡한 것이 아니다.
	- 문제를 해결하는 단계가 명확하면 알고리즘이다.

### 추상 자료형(ADT, Abstract Data Type)

	- 자료구조를 **무엇을 할 수 있는지** 중심으로 정의한 것
	- 실제 구현 방법은 숨기고, 사용할 수 있는 연산만 정의한다.
	- 정보 은닉과 관련이 깊다.

	ADT의 핵심

```text
What: 어떤 기능을 제공하는가?
How : 어떻게 구현하는가는 숨김
```

	예시: 스택 ADT

```text
데이터: LIFO 구조를 유지하는 항목들의 모음

연산
- push(e): 요소 삽입
- pop(): 요소 삭제 및 반환
- peek(): 맨 위 요소 확인
- isEmpty(): 비어 있는지 확인
```

	파이썬 예시

```python
stack = []

stack.append(10)
stack.append(20)

print(stack.pop())
```

	실행 결과

```text
20
```

	시험 포인트

	Q. ADT에서 중요한 것은?

	1. 메모리 주소
	2. 내부 구현 코드
	3. 제공되는 자료와 연산
	4. 실행 파일 크기

	정답: 3
