## 복잡도 분석

### 알고리즘 성능

	알고리즘 성능은 보통 다음 기준으로 평가한다.

	- 실행 시간
	- 메모리 사용량
	- 기능 달성도

	하지만 실제 시간 측정은 환경에 따라 달라질 수 있다.

	시간 측정 예시

```python
import time

start = time.time()

total = 0
for i in range(1000000):
    total += i

end = time.time()

print("실행 시간:", end - start)
```

	주의사항

	- 같은 알고리즘이어도 컴퓨터 성능에 따라 실행 시간이 달라진다.
	- 그래서 일반적으로는 **복잡도 분석**을 사용한다.

---

### 복잡도 분석(Complexity Analysis)

	- 실제 실행하지 않고 알고리즘의 효율성을 분석하는 방법
	- 입력 크기 `n`이 커질 때 실행 시간이 어떻게 증가하는지 본다.

	시간 복잡도

	- 연산 횟수를 입력 크기 `n`에 대한 함수로 표현

	예시

```python
for i in range(n):
    print(i)
```

	이 코드는 `n`번 실행되므로

```text
O(n)
```

---

### 점근적 표기

	입력 크기가 커질 때 증가 속도를 단순하게 나타내는 방법이다.

	대표 표기

| 표기 | 의미 |
|---|---|
| Big-O | 상한 |
| Big-Omega | 하한 |
| Big-Theta | 상한과 하한이 같은 경우 |

	Big-O 예시

```python
for i in range(n):
    print(i)
```

```text
O(n)
```

```python
for i in range(n):
    for j in range(n):
        print(i, j)
```

```text
O(n^2)
```

	자주 나오는 복잡도 순서

```text
O(1) < O(log n) < O(n) < O(n log n) < O(n^2) < O(2^n)
```

	시험 포인트

	Q. 이중 반복문의 일반적인 시간 복잡도는?

```python
for i in range(n):
    for j in range(n):
        print(i, j)
```

	정답: O(n²)

---

### 입력 데이터에 따른 성능

	입력 데이터에 따라 알고리즘의 실행 시간이 달라질 수 있다.

	3가지 경우

	- 최선의 경우: 가장 빨리 끝나는 경우
	- 평균의 경우: 일반적인 경우
	- 최악의 경우: 가장 오래 걸리는 경우

	왜 최악의 경우가 중요한가?

	- 비행기 관제
	- 게임
	- 로보틱스
	- 실시간 시스템

	이런 분야는 최악의 경우에도 일정 시간 안에 끝나야 한다.
