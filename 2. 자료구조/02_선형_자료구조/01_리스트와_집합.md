## 리스트와 집합

### 리스트(List)

	- 항목들이 차례대로 나열된 선형 자료구조
	- 각 항목은 위치 또는 인덱스를 가진다.
	- 임의 위치에 삽입, 삭제, 수정이 가능하다.

	리스트 ADT 주요 연산

```text
Insert(pos, e)   : pos 위치에 e 삽입
Delete(pos)      : pos 위치의 요소 삭제
isEmpty()        : 비어 있는지 검사
getEntry(pos)    : pos 위치 요소 반환
Size()           : 요소 개수 반환
Clear()          : 초기화
Find(item)       : item 위치 찾기
Replace(pos, e)  : pos 위치 값을 e로 변경
Sort()           : 정렬
Append(e)        : 맨 뒤에 추가
```

	파이썬 리스트 예시

```python
arr = [10, 20, 30]

arr.append(40)
arr.insert(1, 15)

print(arr)
```

	실행 결과

```text
[10, 15, 20, 30, 40]
```

	삭제 예시

```python
arr = [10, 20, 30]

value = arr.pop(1)

print(value)
print(arr)
```

	실행 결과

```text
20
[10, 30]
```

### 리스트 구현 방식

### 배열 구조(Array Structure)

	- 같은 자료형의 데이터를 연속된 메모리에 저장
	- 인덱스로 빠르게 접근 가능
	- 항목 접근 시간 복잡도: `O(1)`
	- 중간 삽입/삭제는 비효율적

	배열 구조 느낌

```text
index  0   1   2   3
data  10  20  30  40
```

	장점

	- 인덱스 접근이 빠르다.

	단점

	- 중간 삽입/삭제 시 뒤 요소들을 이동해야 한다.

---

### 연결 구조(Linked Structure)

	- 각 데이터가 다음 데이터의 위치를 링크로 가진다.
	- 메모리상에서 연속되어 있을 필요가 없다.
	- 삽입/삭제가 효율적이다.
	- 특정 위치 접근은 느릴 수 있다.

	연결 구조 느낌

```text
[10 | next] → [20 | next] → [30 | next]
```

	비교표

| 구분 | 배열 구조 | 연결 구조 |
|---|---|---|
| 접근 | 빠름 O(1) | 느림 O(n) |
| 삽입/삭제 | 비효율적 | 효율적 |
| 메모리 | 연속 공간 필요 | 흩어진 공간 가능 |
| 구현 | 비교적 쉬움 | 링크 관리 필요 |

---

### 집합(Set)

	- 원소의 중복을 허용하지 않는다.
	- 원소들 사이에 순서가 없다.

	집합 ADT 주요 연산

```text
Contain(e)     : 원소 포함 여부 검사
Insert(e)      : 원소 삽입
Delete(e)      : 원소 삭제
isEmpty()      : 공집합 검사
Union(setB)    : 합집합
Intersect(setB): 교집합
Difference(B)  : 차집합
Equals(setB)   : 같은 집합인지 검사
Size()         : 원소 개수
```

	파이썬 집합 예시

```python
A = {1, 2, 3}
B = {2, 3, 4}

print(A | B)
print(A & B)
print(A - B)
```

	실행 결과

```text
{1, 2, 3, 4}
{2, 3}
{1}
```

	리스트와 집합 비교

| 구분 | 리스트 | 집합 |
|---|---|---|
| 순서 | 있음 | 없음 |
| 중복 | 허용 | 허용 안 함 |
| 인덱스 접근 | 가능 | 불가능 |
| 사용 예 | 순서 있는 데이터 | 중복 제거 |

	시험 포인트

	Q. 중복을 허용하지 않는 자료구조는?

	정답: 집합(Set)
